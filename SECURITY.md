# 安全与风控说明

## 一、文档目的
本文档用于说明系统当前已落实的安全基线、权限边界、配置安全、考试过程风控策略和当前限制。

## 二、权限安全
- 系统采用 JWT + RBAC 模型
- 后端关键接口通过角色注解进行限制
- 菜单按 `visible_roles` 进行可见性过滤
- 已配置的角色包括：超级管理员、组织管理员、教师、阅卷老师、监考员、考生

## 三、数据安全
- 用户、组织、考试、成绩等核心数据均通过后端持久化表管理
- 数据隔离当前为基础版，主要依赖组织归属字段与角色边界
- 真实数据库密码、AI Key 不得作为正式配置提交
- `.env.local`、本地覆盖配置文件必须被忽略

## 四、考试过程安全
当前已实现：
- 考试时间窗口校验
- 迟到限制基础版
- 参考次数限制基础版
- 提前交卷限制基础版
- 切屏、失焦、退出全屏等行为留痕
- 异常行为触发自动保存
- 监考事件写入数据库并在后台查询

当前未完成但已预留：
- 摄像头监考
- 人脸识别
- 麦克风行为检测
- IP/地理位置风控
- 多设备协同检测

## 五、审计要求
当前已支持：
- 作答提交审计
- 阅卷提交审计
- 后台审计日志查询基础版
- 基础防作弊事件查询基础版

后续仍可增强：
- 用户与权限变更留痕更细化
- 系统参数修改留痕
- 通知发布留痕

## 六、配置安全
- `.env.example` 仅保留示例值
- `application-mysql.yml` 使用环境变量读取 MySQL 参数
- AI 配置使用 `AI_API_BASE_URL`、`AI_API_KEY`、`AI_MODEL`

## 七、运维安全
当前已落实：
- 应用可构建、可打包、可启动
- MySQL 可从空库重建

当前仍需增强：
- 更完整的健康检查说明
- 更完整的异常告警与恢复机制
- 更细的生产运维策略
