# 在线考试系统功能核查矩阵

## 1. 文档说明
### 1.1 核查目标
本次核查面向当前仓库的真实代码、真实页面、真实接口、真实数据库与真实初始化脚本，确认系统到底“已经做到什么、做到什么程度、还缺什么”，为后续补缺口、验收汇报和继续开发提供统一依据。

### 1.2 核查范围
- 根目录正式文档：`README.md`、`PRD.md`、`ARCHITECTURE.md`、`DATA_MODEL.md`、`SECURITY.md`、`PLAN.md`、`IMPLEMENT.md`、`ACCEPTANCE.md`、`EVALS.md`、`DECISIONS.md`、`Documentation.md`、`HANDOFF.md`
- `docs/**`、`database/README.md`、前后端模块 README
- 后端 `controller / service / mapper / entity / security / config`
- 前端 `router / store / api / views / tests/e2e`
- 数据库脚本：`sql/mysql/init.sql`、`backend/src/main/resources/schema.sql`、`backend/src/main/resources/data.sql`
- 本地 MySQL `exam_system` 实库、角色与菜单种子数据

### 1.3 核查方法
1. 逐份读取项目文档，核对历史结论与当前实现是否一致。
2. 核对前端页面、后端接口、数据库表与初始化脚本之间的对应关系。
3. 连接本地 MySQL 校验核心表、角色、菜单、考试、答卷、事件等真实数据。
4. 执行前端构建、后端测试与 Playwright 浏览器回归，确认主链路是否可跑。
5. 对每个能力点给出判定、依据、缺口与影响说明。

## 2. 判定标准
| 判定 | 含义 |
|---|---|
| 已实现 | 页面、接口、表结构、主流程已打通，并已在本轮核查中找到明确实现依据。 |
| 基础版实现 | 已有真实落地，但规则、范围、体验或治理能力仍明显偏基础。 |
| 部分实现 | 只覆盖了能力的一部分，或者只有局部链路可用，仍不足以宣称完整支持。 |
| 仅预留/占位 | 只保留了字段、配置项、入口或文档说明，未形成可用业务能力。 |
| 未实现 | 当前仓库、页面、接口、表结构中均未发现对应实现。 |
| 当前无法确认 | 只有在当前代码、脚本和环境都无法得出结论时使用；本轮核查未使用该状态。 |

## 4. 结论汇总
### 4.1 总体结论
- 当前系统已经形成“题库 -> 试卷 -> 考试发布 -> 学生作答 -> 阅卷 -> 成绩分析 -> 基础监考留痕”的一期核心闭环。
- 当前更适合定义为“核心闭环版 + 多项高规格能力仍未补齐”的状态，而不是完整的高规格在线考试平台。
- 本轮核查额外发现并修复了一个高优先级一致性缺口：学生可直接输入未授权路由访问监考页，前端会展示页面骨架但接口返回 403。现已通过前端路由守卫与 Playwright 回归补齐。

### 4.2 状态统计
- 已实现项统计：38 项
- 基础版实现项统计：35 项
- 部分实现项统计：22 项
- 仅预留/占位项统计：6 项
- 未实现项统计：71 项

### 4.3 需要优先补齐的项
1. 权限一致性与数据权限继续收口，避免任何“菜单隐藏但路由可进”“路由可进但接口 403”的半残状态。
2. 题库题型扩展与富媒体能力，至少补齐填空、论述、附件、多媒体基础支持。
3. 组卷规则增强，补知识点维度与更精细的考试发布策略。
4. 学生侧成绩查看、错题分析、考后反馈闭环。
5. 安全与运维能力，包括登录风控、限流、健康检查、备份恢复与并发压测。
