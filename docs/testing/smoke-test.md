# 烟雾测试说明

## 一、目标
验证系统构建、数据库回归、关键接口和关键浏览器流程是否正常。

## 二、后端验证
1. `mvn -q -DskipTests compile`
2. `mvn -q test`
3. `mvn -q -DskipTests package`

## 三、前端验证
1. `npm.cmd run build`
2. `npm.cmd run test:e2e`

## 四、数据库回归
1. 删除并重建 `exam_system`
2. 导入 `sql/mysql/init.sql`
3. 检查核心表与种子数据
4. 用 MySQL 模式启动后端

## 五、关键接口 smoke
- 管理员登录
- 组织列表
- 用户列表
- 审计日志列表
- 配置项与字典项列表
- 学生消息列表
- 待考列表
- 考试口令进入工作区
- 分析概览

## 六、浏览器级 E2E
1. 学生登录 -> 进入考试 -> 作答 -> 提交
2. 阅卷老师登录 -> 打开阅卷任务 -> 评分 -> 提交

## 七、预期结果
- 所有自动化测试通过
- 所有关键接口返回成功
- MySQL 模式下工作区可正常读取题目
