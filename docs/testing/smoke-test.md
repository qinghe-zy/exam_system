# 烟雾测试说明

## 一、目标
验证系统构建、数据库回归、关键接口和关键浏览器流程是否正常。

## 二、后端验证
1. `mvn -q -DskipTests compile`
2. `mvn -q test`
3. `mvn -q -DskipTests package`

## 三、前端验证
1. `npm.cmd run build`
2. `npm.cmd run test:e2e`

## 四、数据库回归
1. 删除并重建 `exam_system`
2. 导入 `sql/mysql/init.sql`
3. 检查核心表与种子数据
4. 用 MySQL 模式启动后端

## 六、浏览器级 E2E
1. 学生登录 -> 进入考试 -> 作答 -> 提交
2. 阅卷老师登录 -> 打开阅卷任务 -> 评分 -> 提交
3. 学生消息中心 -> 我的成绩 -> 成绩详情
4. 学生答卷回看中心 -> 答卷回看 / 错题本
5. 学生直达未授权监考页 -> 路由守卫拦回首页
6. 学生注册 -> 登录 -> 找回密码 -> 新密码登录
7. 教师创建填空题 / 材料题 -> 按知识点自动组题
8. 严格考试态下触发复制 / 粘贴 / 右键 / 快捷键拦截 -> 监考事件列表可查
9. 管理员进入登录风险记录页 -> 可查看成功/失败登录及风险说明
10. 管理员消息中心 -> 可查看 `SECURITY_ALERT` 站内告警
11. 阅卷老师评分 -> 复核通过 -> 学生提交申诉 -> 管理端打开申诉处理抽屉
12. 教师进入分析页 -> 打开质量报告抽屉 -> 导出质量报告 Markdown

- 所有自动化测试通过
- 所有关键接口返回成功
