# 基础压测与性能说明

## 一、文档目的
本文档用于说明当前阶段的基础压测范围、执行脚本、指标口径、结果记录方式和当前限制。目标不是替代专业压测平台，而是建立一个可重复执行、可对比、可交付的性能基线。

## 二、当前压测范围
当前版本先覆盖登录接口基础压测：
- 接口：`POST /api/auth/login`
- 目标：
  - 验证接口在本地开发环境下的基础响应能力
  - 观察登录限流与锁定策略是否正常生效
  - 形成后续性能对比基线

## 三、压测脚本
脚本位置：
- `scripts/load-test-login.ps1`

标准命令：
```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File scripts/load-test-login.ps1 `
  -BaseUrl http://localhost:8083 `
  -Username 900001 `
  -Password 123456 `
  -TotalRequests 40 `
  -DelayMilliseconds 100
```

## 四、输出指标
脚本会输出以下指标：
1. 成功请求数
2. 失败请求数
3. 平均耗时
4. P95 耗时
5. 最大耗时
6. 失败信息分布

## 五、执行建议
### 5.1 执行前
1. 确认后端已启动。
2. 确认本地 MySQL 可用。
3. 确认测试账号存在。

### 5.2 执行时
1. 先以正确密码执行一轮基线压测。
2. 再使用错误密码执行一轮风控压测，观察锁定与限流是否生效。
3. 压测期间同步观察：
   - 登录风险记录页
   - 管理员消息中心

### 5.3 执行后
1. 记录平均耗时和 P95。
2. 记录是否触发风控拦截。
3. 将结果同步到 `EVALS.md` 与 `Documentation.md`。

## 六、当前阶段限制
1. 当前脚本属于单机本地基础压测，不代表正式生产容量。
2. 当前未引入 JMeter、Locust、k6 等专业压测平台。
3. 当前未覆盖考试工作区保存、交卷、分析接口等高负载链路。
4. 当前未覆盖弱网、移动端、多浏览器矩阵和并发考生大规模场景。

## 七、建议口径
当前应将其定义为“性能基线采样”，而不是“生产容量证明”。后续若进入更高规格验收，应补：
1. 登录接口并发压测
2. 学生作答保存链路压测
3. 交卷与阅卷链路压测
4. 数据库慢 SQL 分析
5. 监控与告警联动
