\# AGENTS.md



\## 非功能性硬要求

必须持续关注：

\- 权限边界

\- 数据隔离

\- 可扩展性

\- 可维护性

\- 审计与日志

\- 可测试性

\- 可部署性

\- 安全性



\## 实施边界

允许：

\- 自主创建目录、模块、脚本、文档

\- 自主抽取共享组件与公共能力

\- 自主补充合理但缺失的实现细节

\- 自主增加测试、校验、runbook、初始化脚本



不允许：

\- 未经论证地更换技术栈

\- 为了省事把敏感信息直接硬编码提交

\- 未验证就声称可运行

\- 只做空目录和空 README 就视为完成

\- 写死未来扩展空间

\- 引入与项目目标无关的框架和依赖



\## 文档同步规则

每完成一个里程碑，必须同步更新：

\- PLAN.md

\- Documentation.md

\- DECISIONS.md

\- CHANGELOG.md

\- 对应模块 README / runbook / SQL 说明



\## 验证规则

每一阶段至少执行：

\- 结构检查

\- 依赖安装检查

\- 后端构建/测试（如可行）

\- 前端构建/测试（如可行）

\- 数据库脚本检查

\- 路由/权限/配置一致性检查



\## 安全规则

\- 禁止把真实生产密码、token、私钥提交进仓库。

\- 本地开发配置应使用 .env.local、application-local.yml 或同类本地覆盖文件，并确保被 .gitignore 忽略。

\- 若必须在文档中展示配置，仅使用示例值。



\## 交付标准

项目只在以下条件满足时才可宣称“阶段完成”：

1\. 代码结构清晰

2\. 文档已同步

3\. 数据模型有依据

4\. 关键链路可解释

5\. 验证结果真实可追溯

6\. 未解决问题已写明

\## 本地数据库约束

当前本地开发数据库已经初始化完成。



AI 在开发过程中可以按需检查、修改、补充数据库结构与初始化脚本，但必须遵守以下规则：

\- 数据库类型：MySQL

\- host：127.0.0.1

\- port：3306

\- username：root

\- password：123456



执行要求：

\- AI 可以在本地开发环境中使用上述凭据连接数据库进行建库、建表、改表、初始化与验证。

\- 若需要调整数据库名、补充表结构、更新初始化脚本，必须同步更新文档与 SQL 脚本。

\- 不要把真实数据库凭据硬编码提交到生产配置中。

\- 优先通过 `.env.local`、`application-local.yml`、本地覆盖配置等方式使用该连接信息。

\- 若为本地开发方便临时写入配置，必须确保对应文件被 `.gitignore` 忽略，或在提交前改回示例值。



\## AI API Key 约束

项目后续可能接入 AI 能力，但当前阶段仅需预留接入能力，不直接写入真实 key。



要求：

\- 只预留配置项、接口适配层、调用边界与占位文档。

\- 不得提交真实 API Key。

\- 在 `.env.example` 或等价示例配置中保留类似以下字段：

&#x20; - `AI\_API\_BASE\_URL=`

&#x20; - `AI\_API\_KEY=`

&#x20; - `AI\_MODEL=`

\- 若项目中设计 AI 相关模块，必须通过环境变量读取配置。



\## Git 远端仓库目标

本项目最终代码应导入并推送到以下远端仓库：

\- https://github.com/qinghe-zy/exam\_system.git



要求：

\- AI 在本地完成结构、代码、文档、脚本、数据库初始化与验证后，应将仓库 remote 指向该地址。

\- 若远端未配置，则添加为 `origin`。

\- 若远端配置错误，则修正并在 `Documentation.md` 与 `HANDOFF.md` 中记录。

\- 目标是让最终代码在远端仓库可见。

\- 若推送失败，必须写出精确失败命令、错误信息与后续人工 push 步骤。

\## 输出风格

1\. 当前阶段

2\. 已完成内容

3\. 本轮改动文件/模块

4\. 验证结果

5\. 剩余风险

6\. 下一步建议
