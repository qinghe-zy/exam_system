# 验收标准

## 1. 文档目的
本文档用于定义当前项目的硬验收口径。自本轮起，不再以“核心考试闭环完成”作为最终完成标准，而以“功能补全到接近《PRD》目标口径并完成真实验证”为验收基准。

## 2. 当前硬验收原则
### 2.1 总体原则
以下情况均不算完成：
- 只有表结构，没有真实流程
- 只有页面，没有接口或权限
- 只有接口，没有前端入口
- 只有预留字段、预留按钮或占位文档
- 只有 mock，没有真实主流程
- build 通过但关键链路未验证
- 文档声称支持，但代码、数据库或测试不支撑

### 2.2 当前完成口径
当前项目只有在以下条件尽可能满足后，才可判定为“阶段完成”：
1. 12 大模块能力核查已完成
2. 核查矩阵中原本标记为“基础版实现 / 部分实现 / 仅预留 / 未实现”的能力项，已被重新实现、补强或明确列入 blocker
3. 代码、数据库、初始化脚本、接口、页面、权限、文档和验证结果保持一致
4. 关键主流程已形成完整闭环
5. 已建立并执行尽可能完整的测试体系
6. 浏览器回归与 Playwright 已尽可能覆盖核心链路
7. 交付文档已同步为最终真实状态

## 3. 当前必须补到“完成或高质量基础版”的能力域
1. 账号与权限增强
2. 题库与题型增强
3. 试卷与考试规则增强
4. 学生端考后闭环增强
5. 监考与防作弊增强
6. 阅卷与成绩治理增强
7. 通知与协同增强
8. 安全与运维增强

## 4. 分能力域验收要求
### 4.1 账号与权限增强
至少应满足：
- 注册基础版
- 找回密码基础版
- 邮箱/短信验证码基础版
- 按钮级权限基础版
- 更严格的数据隔离一致性

验收证据：
- 页面
- 接口
- 权限点
- 数据隔离验证
- 测试记录

### 4.2 题库与题型增强
至少应满足：
- 填空题
- 论述题
- 材料题
- 富文本
- 图片/附件基础支持
- 题目使用统计
- 查重/去重基础版
- 知识点自动组题基础版

验收证据：
- 题目录入页
- 题目持久化结构
- 组卷可用性
- 学生端可展示/作答

### 4.3 试卷与考试规则增强
至少应满足：
- 更复杂的策略组卷
- 知识点/章节/分值比例抽题
- A/B 卷或平行卷基础版
- 补考/缓考/重考
- 批次安排
- 签到规则
- 准考证/通知单基础版

### 4.4 学生端考后闭环增强
至少应满足：
- 设备检测基础版
- 标记题目/待复查
- 学生成绩查询
- 考后答案/解析查看策略
- 个人答卷回看
- 错题沉淀基础版
- 防刷新/防误触增强

当前阶段已具备的验收证据补充：
- 进入严格考试态前会执行设备检测
- 设备检测不通过时可阻止进入考试工作区，并展示失败原因
- 考试工作区待复查标记与保存持久化
- `答卷回看` 页面可查看完整答卷
- `错题本` 页可按已发布成绩聚合错题
- 离页确认可拦截误刷新/误离开

### 4.5 监考与防作弊增强
至少应满足：
- 复制粘贴限制
- 右键/快捷键限制
- 多设备登录限制基础版
- 登录风险记录基础版
- IP/设备信息留痕基础版
- 监考策略配置中心基础版

当前阶段已具备的验收证据补充：
- 严格考试态下复制 / 粘贴 / 右键 / 高风险快捷键拦截
- 设备上下文与客户端 IP 写入监考事件
- 配置中心存在 `anti_cheat` 分组配置项
- 监考事件页可查看新增设备与 IP 字段
- 管理端可查看登录风险记录
- 严格考试态下可阻止同一考试切换到另一设备

### 4.6 阅卷与成绩治理增强
至少应满足：
- 复核
- 重判
- 申诉流程基础版
- 更完整的阅卷任务状态
- 成绩单导出
- 分析报表导出
- 考试质量报告基础版

当前阶段已具备的验收证据补充：
- 主观题评分完成后，答卷会先进入 `REVIEW_PENDING`，需人工复核后才发布
- 阅卷工作区支持“复核通过并发布”和“退回重判”
- 学生可在成绩详情页提交申诉，并查看申诉历史
- 管理端可在成绩中心查看并处理申诉，当前支持“驳回”与“转入重判”
- 重判后可重新提交阅卷并再次复核发布
- 分析页当前可查看正式质量报告，并导出 Markdown 质量报告

### 4.7 通知与协同增强
至少应满足：
- 开考前提醒
- 成绩发布提醒增强
- 通知模板
- 至少一个真实可用通知通道或可运行 mock 通道

### 4.8 安全与运维增强
至少应满足：
- 登录失败风控基础版
- 限流/防爆破基础版
- 健康检查
- 备份恢复说明与脚本基础版
- 异常告警 runbook
- 基础压测说明

当前阶段已具备的验收证据补充：
- 同一账号连续失败达到阈值后会被临时锁定，找回密码成功后可自动解锁
- 同一客户端 IP 在限流窗口内请求过多时会被认证接口拦截
- 同一验证码目标已具备冷却时间和窗口次数限制
- 触发锁定或 IP 限流时，管理员与机构管理员会收到 `SECURITY_ALERT` 站内消息
- 已提供 `scripts/backup-mysql.ps1`、`scripts/restore-mysql.ps1`、`scripts/load-test-login.ps1`
- 已提供 `docs/runbooks/mysql-backup-restore.md`、`docs/runbooks/security-alert-runbook.md`、`docs/testing/performance-baseline.md`

## 5. 测试与验证验收项
### 5.1 基础构建与运行验证
- 后端 compile / test / package
- 前端 install / build
- 数据库初始化脚本回归
- 系统启动验证
- 核心接口可访问验证

### 5.2 主流程验证
- 管理员主流程
- 教师/命题人主流程
- 阅卷老师主流程
- 监考员主流程（若当前支持）
- 学生主流程

### 5.3 业务流程验证
- 题目录入 -> 组卷 -> 试卷保存 -> 考试发布 -> 学生进入考试 -> 作答 -> 自动保存 -> 提交 -> 阅卷 -> 成绩 -> 分析
- 随机组卷与策略组卷
- 考试规则生效
- 权限一致性
- 防作弊事件触发、落库、查询、展示
- 组织/班级/用户/角色管理
- 通知/消息相关流程

### 5.4 自动化验证
- Playwright E2E
- 权限矩阵测试
- 接口自动化 smoke / 回归测试
- 数据库初始化回归检查脚本

## 6. 最终验收必须输出
1. 完整的功能补全清单
2. 补全前后对比表
3. 逐模块实现说明
4. 新增表结构与字段说明
5. 新增接口说明
6. 新增页面与路由说明
7. 权限点变化说明
8. 测试与验证结果
9. 剩余无法完成项与真实 blocker
