# 验收标准

## 一、文档目的
本文档用于说明当前项目的验收标准，避免出现“只有页面、没有链路”“只有 SQL、没有业务”“只有文档、没有实现”的假完成情况。

## 二、功能验收项
### 1. 用户与权限
- 支持登录
- 支持角色边界区分
- 支持组织基础管理
- 支持用户基础维护
- 支持考生批量导入基础版

### 2. 题库管理
- 支持主流题型
- 支持题干、选项、答案、解析、知识点、标签、审核状态、版本号
- 支持题目新增、编辑、删除、导入、导出基础版

### 3. 试卷与组卷
- 支持手工组卷
- 支持随机组卷基础版
- 支持按难度策略组卷基础版
- 支持分值配置与总分计算

### 4. 考试发布与作答
- 支持考试计划创建
- 支持绑定试卷与考生
- 支持时间窗口与基础规则校验
- 支持考生查看待考、进入考试、保存、交卷
- 支持答题卡与自动保存基础版

### 5. 阅卷与评分
- 支持客观题自动判分
- 支持主观题人工评分
- 支持成绩记录回写

### 6. 成绩与分析
- 支持成绩查看
- 支持平均分、最高分、最低分、及格率
- 支持排名、分数段、知识点、题目得分率基础分析

### 7. 防作弊与审计
- 支持基础考试事件留痕
- 支持后台查看反作弊事件
- 支持查看审计日志基础版

## 三、数据库验收项
- 能删除并重建 `exam_system`
- 能从 `sql/mysql/init.sql` 完整导入
- 核心表、角色、菜单、组织、题库、考试、答卷、成绩、事件数据存在
- MySQL 模式启动后关键接口可正常读取数据库

## 四、文档验收项
- 全部正式交付文档为中文主文档
- 关键文档必须详细、科学、可交付
- 文档与代码、数据库、接口、脚本、验证结果一致
- 每份文档整改状态可追踪

## 五、不算完成的情况
- 只有目录没有实现
- 只有页面没有数据库与接口支撑
- 只有接口没有主流程验证
- 只有 SQL 没有系统运行验证
- 只有中文标题没有中文正文
- 只有提纲没有详细说明
