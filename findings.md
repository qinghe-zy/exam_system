# Findings

## 本轮关键发现
1. 当前系统已形成一期核心闭环，不再是只有后台 CRUD 的轻量模板。
2. 当前最大产品差距集中在：
   - 复杂题型与多媒体题
   - 学生考后闭环
   - 正式监考与风控
   - 安全与运维能力
3. 前端存在权限一致性缺口：
   - 学生可直接输入 `/exam/proctor`
   - 页面骨架能打开
   - 后端接口返回 403
   - 页面最终出现“Access Denied”半残状态
4. 现有 Playwright 中存在时间写死问题，部分用例会随着当天时间流逝自然失效。

## 本轮处理结果
1. 已在前端路由增加基于菜单树的访问守卫
2. 已新增权限回归用例
3. 已将关键 E2E 改为按当前时间动态生成考试时间窗口
4. 已新增系统能力核查矩阵和差距分析正式文档

## 剩余主要差距
1. 题型覆盖仍偏少
2. 学生成绩查看与错题闭环未实现
3. 防作弊仍是基础版
4. 安全与运维仍缺健康检查、限流、告警、备份恢复与压测
